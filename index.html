Here is the step by step psuedo code for you to follow:

Open up your top quality linux computer!

First create a new remote repository (which is a folder basically) on github.com and name it. It will help track every change you make with your program/website.

Copy SSH code of new repository (or repo for short) and open the linux local terminal

keyboard keys to press: control + alt +t

control + alt +t opens the terminal.

In the command line of the terminal put:

git clone >SSH code pasted.

You can check the clone of the repo by entering in the command line:

cd >name of repo.

git remote -v

You should see 2 URLs and this confirms you can save all changes you make and access them again.


Next, in your local terminal, while in your repo, use the following touch command to create the first file:

touch README.md

A README file contains information about other files in a directory or archive of computer software. A form of documentation, it is usually a simple plain text file called Read Me , READ.ME , README.

Start working on all the files for the program from HTML and CSS to Javascript.



To add all files or some files to remote repo for safekeeping and monitoring, use the following commands:

git add. (stages all files)

git add >specified names of files. (stages specific files)

git status (shows unstaged files in red and staged files in green)

If you re-edit the files, you must re-add them.

git commit -m ">file name" (add an extra space in the quotes and write more file names to commit more than one at a time.

If you re-add edited files when committing, a short description in the quotes next to the file about your edit should be written, such as 'changed sentence >file name... etc

git log (verifies files are in commit phase, ready to be put in the github.com repo you created.

git push origin master (sends your files to github.com for safe keeping.

git status (will verify your files existing locally and on the internet. :)
